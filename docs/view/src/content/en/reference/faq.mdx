## Development Questions

<AccordionGroup>
  <Accordion title="How do I add a new tool?" icon="Wrench" client:load>
    1. Define the tool in `server/main.ts` using `createTool`
    2. Add it to the `withRuntime` tools array
    3. Run `npm run gen` to update types
    4. Test it via the deco UI or RPC client
  </Accordion>

  <Accordion title="How do I connect to an external API?" icon="Globe" client:load>
    1. Add the integration via the deco dashboard
    2. Run `npm run gen` to get typed interfaces
    3. Use `env.INTEGRATION_NAME.method()` in your tools
  </Accordion>
</AccordionGroup>

## Technical Questions

<AccordionGroup>
  <Accordion title="What's the difference between tools and workflows?" icon="Settings" client:load>
    Tools are single functions that perform specific tasks. Workflows orchestrate multiple tools with logic, conditions, and loops.
  </Accordion>

  <Accordion title="Can I use my own database?" icon="Database" client:load>
    Yes. You can use Cloudflare D1, Workers KV, or connect to external databases like 
    Supabase or Planetscale via HTTP APIs.
  </Accordion>
</AccordionGroup>

## Integration Questions

<AccordionGroup>
  <Accordion title="Which integrations are available?" icon="Link" client:load>
    Many common services like Slack, Gmail, Notion, Stripe, and more. Check the deco dashboard for the full list.
  </Accordion>

  <Accordion title="Can I create custom integrations?" icon="Code" client:load>
    Yes, you can write custom tools that call any HTTP API using `fetch`.

## Troubleshooting

<AccordionGroup>
  <Accordion title="My tool isn't working" icon="AlertCircle" client:load>
    1. Check that it's registered in `withRuntime`
    2. Verify your Zod schemas
    3. Check the logs for errors
    4. Ensure you ran `npm run gen` after changes
  </Accordion>

  <Accordion title="My workflow isn't executing" icon="AlertTriangle" client:load>
    1. Make sure you called `.commit()` at the end
    2. Check that all steps are properly chained
    3. Verify your input/output schemas match
  </Accordion>
</AccordionGroup>
